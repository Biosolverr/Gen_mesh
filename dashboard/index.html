Один сквозной прогон композитной задачи через полную сеть контрактов: Registry → Coordinator → Agent IC(s) → Aggregator. Каждый шаг ниже — реальная транзакция или view-вызов в модели GenLayer, данные соответствуют контрактам, реализованным на этапах 2–5.

Read-only режим: подтягивает реальное состояние уже задеплоенных контрактов через genlayer-js view-вызовы. Приватные ключи в браузере не используются — только readContract. Отправка задач (submit_task) выполняется отдельно через GenLayer Studio / CLI, здесь только наблюдение за результатом.
